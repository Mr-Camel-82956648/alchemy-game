# AI法阵·炼金术士

当前项目采用前后端分离结构：

- `frontend/`：现有前端玩法、战斗逻辑、法阵展示
- `backend/`：AI 生成与结构化接口（首版可用 mock）
- `docs/`：当前有效文档

## 当前阶段目标
1. 保留现有前端玩法
2. 建立统一法阵/怪物数据结构
3. 引入极简属性与伤害规则
4. 新增最小后端 forge 接口
5. 实现前后端通信

## 重要原则
- 视觉优先，数值为视觉翻译
- 主属性参与战斗，副属性只记录
- 老法阵兼容迁移，新法阵优先结构化
- 资源一律通过稳定 id 管理，不通过数组位置管理
